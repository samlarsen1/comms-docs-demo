The **Comms Platform API** provides a unified interface for delivering communications
across multiple channels — email, SMS, push notifications, and documents (both digital
and postal).

## Key Concepts

### Channels
| Channel | Description |
|---|---|
| `email` | Transactional and marketing emails |
| `sms` | SMS to mobile devices |
| `push` | Mobile push notifications |
| `document_digital` | PDFs delivered via secure online portal |
| `document_postal` | Physical mail to a postal address |
| `auto` | Smart routing — platform picks the best channel |

### Recipient Identification
Recipients can be identified in two ways:
- **Known parties** — identified by a `partyId`. Contact details and channel preferences
  are resolved automatically from the party profile.
- **Unknown parties** — contact details supplied directly in the request (e.g. an email
  address or phone number). No party record is required.

### Smart Routing
Set `channel: auto` to let the platform determine the optimal delivery channel based
on the recipient's stored preferences and available contact details.

### Idempotency
Supply an `idempotencyKey` in `metadata` to safely retry requests without risk of
duplicate sends.

## Base URL
All endpoints are relative to `https://api.comms.example.com`.

## Authentication
All requests require a Bearer token in the `Authorization` header:
```
Authorization: Bearer <your-token>
```
