# Client

|---|---|

Registered upstream client application authorised to produce CommunicationEvents. Clients are onboarded and assigned an applicationId before they can publish to the communications platform. Each client may own one or more Workloads.

## Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `clientId` | `string` | Yes | — | Bank-assigned UUID uniquely identifying this client registration. |
| `applicationId` | `string` | Yes | — | Unique application code assigned at onboarding. Must match pattern ^AL\d&#123;6&#125;$ — prefix 'AL' followed by exactly 6 digits, e.g. 'AL000123'. Validated at registration time; the communications platform will reject events from unregistered application IDs. |
| `applicationName` | `string` | Yes | — | Human-readable name of the upstream application, e.g. 'Core Banking System', 'Fraud Engine', 'Card Processing'. |
