# Workload

A named workload belonging to a registered Client. Workloads represent distinct communication use cases or integration streams within a client application, e.g. 'MONTHLY_STATEMENTS', 'FRAUD_ALERTS', 'INTEREST_RATE_CHANGES'. Every CommunicationEvent must carry a workloadId to enable volume attribution, SLA tracking, and billing back to the originating workload.

## Fields

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `workloadId` | `string` | Yes | — | Bank-assigned UUID uniquely identifying this workload. |
| `clientId` | `string` | Yes | — | Reference to the Client that owns this workload. |
| `workloadName` | `string` | Yes | — | Short human-readable name for this workload, e.g. 'Monthly Current Account Statements'. |
| `workloadDescription` | `string` | Yes | — | Full description of the workload's purpose, expected communication types, and typical volume. |
