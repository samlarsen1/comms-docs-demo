# AutoChannelDispatch

|---|---|

| **Compatibility** | `BACKWARD` |
| **Status** | `ACTIVE` |
| **Source** | [schemas/channels/AutoChannelDispatch.avsc](https://github.com/samlarsen1/comms-docs-schemas/blob/main/schemas/channels/AutoChannelDispatch.avsc) |

Describes the outcome of an AUTO channel dispatch attempt. When a CommunicationEvent has channel=AUTO, the dispatcher resolves the delivery channel(s) at runtime by: (1) loading the customer's CommunicationPreferences for the given eventType and productType; (2) filtering channels where ChannelConsent is given; (3) for each preferred channel in order, checking whether an ACTIVE Template exists that supports the payload type and channel; (4) attempting delivery and cascading to the next channel on failure if cascade is enabled. This record captures the full resolution and attempt history for audit, debugging, and retry purposes.

## Nested Types

### CascadeConfig {#cascadeconfig}

Configuration governing cascade behaviour for this AUTO dispatch.

| Field | Type | Required | Default | Description |
|-------|------|----------|---------|-------------|
| `stopOnFirstSuccess` | `boolean` | No | `true` | If true, cascade stops after the first successful delivery. If false, the event is delivered to all available channels (e.g. for regulatory documents that must be delivered everywhere). |
| `respectCustomerPreferences` | `boolean` | No | `true` | If true, the customer's CommunicationPreferences preferredChannelOrder is used to determine cascade order. If false, the fallbackChannelOrder is used directly. |
| `fallbackChannelOrder` | array of [`ChannelType`](./common/enums/ChannelType) | No | `["EMAIL","SMS","PUSH","POST"]` | Channel order to use if respectCustomerPreferences is false, or if the customer has no preferences set for this event type. Must not contain AUTO. |
| `requireTemplateMatch` | `boolean` | No | `true` | If true, a channel is skipped if no active template exists for the payload type on that channel. If false, dispatches without a template (raw payload delivery). |
| `maxCascadeDepth` | `int` | No | `3` | Maximum number of channels to attempt in the cascade. Prevents excessive delivery attempts. |
| `retryAttemptsPerChannel` | `int` | No | `2` | Number of retry attempts per channel before cascading to the next. |
| `retryDelaySeconds` | `int` | No | `60` | Delay in seconds between retry attempts on the same channel. |
| `skipOptedOutChannels` | `boolean` | No | `true` | If true, channels for which the customer has not given consent are silently skipped. If false, an OPTED_OUT attempt is logged. |
